# Phase 01 — MCP Server Layer

## الهدف

بناء طبقة السيرفر الأساسية: نقطة الدخول التي تستقبل اتصال عميل MCP (مثل Cursor) وتعرّفه بالسيرفر وقدراته، دون تنفيذ أدوات أو مهارات أو إضافات بعد.

---

## المخرجات المتوقعة

- عملية MCP Server قابلة للتشغيل عبر stdio (أو النقل المختار)
- تسجيل السيرفر وعرض اسمه وقدراته للعميل
- استجابة لطلبات `initialize` و `initialized` حسب المواصفة
- هيكلية تسمح لاحقاً بإضافة الأدوات (Phase 02) والمهارات (Phase 03) والإضافات (Phase 04)

---

## المهام الفرعية

### 1.1 اختيار مكتبة MCP وطريقة النقل

- [ ] تثبيت واستخدام مكتبة السيرفر (مثلاً `@modelcontextprotocol/sdk`)
- [ ] اختيار النقل: **stdio** (الافتراضي لـ Cursor وغيره) أو SSE إذا مطلوب
- [ ] إنشاء نقطة الدخول الرئيسية (مثلاً `src/server/index.ts`) التي:
  - تقرأ من stdin وتكتب إلى stdout عند استخدام stdio
  - لا تطلق أي أدوات أو مهارات بعد

### 1.2 تهيئة السيرفر (Initialize)

- [ ] تنفيذ معالجة طلب `initialize`:
  - إرجاع `serverInfo`: اسم السيرفر وإصداره (مثلاً `company-mcp-hub`, `0.1.0`)
  - إرجاع `capabilities` حسب المواصفة (مثلاً قائمة الأدوات لاحقاً)
- [ ] إرسال استجابة `initialized` بعد نجاح التهيئة
- [ ] توثيق الإصدار واسم السيرفر في مكان واحد (مثلاً `src/config/constants.ts` أو `package.json`)

### 1.3 هيكلية تسجيل الأدوات (للاستخدام لاحقاً)

- [ ] تعريف واجهة أو نوع (في `src/types/`) لوصف أداة واحدة:
  - الاسم، الوصف، مخطط المدخلات (input schema)
- [ ] تحضير آلية تسجيل (registry أو قائمة) للأدوات في السيرفر — يمكن أن تكون فارغة أو تسجّل أداة تجريبية واحدة للتحقق
- [ ] عند طلب `tools/list` من العميل: إرجاع القائمة المسجّلة (حتى لو فارغة أو تجريبية)
- [ ] عند طلب `tools/call`: التحضير لمعالج (handler) يستقبل اسم الأداة والوسائط — التنفيذ الفعلي في Phase 02

### 1.4 إدارة دورة الحياة

- [ ] التعامل مع إغلاق العميل (إغلاق stdin أو إشارة إنهاء) وإيقاف السيرفر بشكل نظيف
- [ ] تجنب تسريب الموارد (عدم ترك عمليات أو مقابس مفتوحة)

### 1.5 التشغيل والاختبار

- [ ] سكربت تشغيل: `npm run start` أو `node dist/index.js` يطلق السيرفر ويستمع على stdio
- [ ] اختبار يدوي أو آلي: عميل MCP يرسل `initialize` ويستلم الرد الصحيح و`tools/list` يعيد قائمة (فارغة أو تجريبية)

---

## معايير الإكمال

- العميل (مثل Cursor) يمكنه الاتصال بالسيرفر والحصول على تهيئة ناجحة
- `tools/list` يعيد بنية صحيحة (حتى لو بدون أدوات حقيقية بعد)
- لا حاجة لأدوات فعلية أو مهارات أو إضافات خارجية في هذه المرحلة

## الملفات المقترحة

| الملف | الغرض |
|-------|--------|
| `src/server/index.ts` | نقطة الدخول، إنشاء السيرفر وربط النقل |
| `src/server/server.ts` أو `server.js` | منطق تهيئة MCP ومعالجة الطلبات الأساسية |
| `src/types/tools.ts` (أو مشابه) | أنواع واجهة الأداة ومخطط المدخلات |
| `src/config/constants.ts` (اختياري) | اسم السيرفر، الإصدار، القدرات الافتراضية |

## ملاحظات
